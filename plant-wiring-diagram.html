2. NodeMCU pinout — only the pins we use

NodeMCU silkscreen labels (D1, A0, etc.) are different from the underlying ESP8266 GPIO numbers. Use the silkscreen names in code; the Arduino core maps them for you.

SilkscreenGPIOUsed forWhy this pin
3V3—Power → Soil sensor VCC3.3 V regulated rail, ~600 mA available.
Vin—Power → Relay VCC~5 V from USB. Relay coil needs more than 3V3.
GND—Common groundTie sensor GND, relay GND, and pump-PSU (−) here.
A0ADC0Soil sensor AOUTOnly analog input. Has on-board divider, safe to 3.3 V.
D1GPIO5Relay IN (pump control)Boot-safe, no strap-pin issues. Avoid D3/D4/D8.

Sanity check before powering on:

  1. Multimeter on continuity mode → buzz between NodeMCU GND and pump PSU (−). If no buzz, common-ground wire is missing.
  2. Pump (+) should not have continuity to PSU (+) when the relay is idle — only when D1 is asserted.
  3. 1N4007 stripe (cathode) is on the positive side of the pump. Backwards = diode short = burnt diode.

5. How to flash the code

Step 1 — Install drivers (one-time)

Windows usually auto-detects, but if Device Manager doesn't show a new COM port when you plug the NodeMCU in:

  • NodeMCU v1.0 (Amica) uses the CP2102 USB-serial chip → install Silicon Labs CP210x VCP driver.
  • NodeMCU v3 (LoLin) uses the CH340/CH341 chip → install the CH340 driver.

After install, plug the board in and check Windows Device Manager → Ports (COM & LPT) — you'll see something like USB-SERIAL CH340 (COM5). Note the COM number.

Step 2 — Install Arduino IDE + ESP8266 board package

  1. Install Arduino IDE 2.x from arduino.cc (or 1.8.19 if you prefer).
  2. Open File → Preferences. In Additional Boards Manager URLs, paste:
    http://arduino.esp8266.com/stable/package_esp8266com_index.json
  3. Open Tools → Board → Boards Manager, search esp8266, install "esp8266" by ESP8266 Community.
  4. Select Tools → Board → ESP8266 Boards → NodeMCU 1.0 (ESP-12E Module).
  5. Set Tools → Upload Speed → 115200, CPU Frequency → 80 MHz, Flash Size → 4MB (FS:2MB OTA:~1019KB).
  6. Set Tools → Port to the COM number from Step 1.

Step 3 — Install libraries

Open Tools → Manage Libraries and install:

  • Universal Telegram Bot by Brian Lough — latest.
  • ArduinoJson by Benoit Blanchon — pin to version 6.x. v7 is not yet compatible with the Telegram library.

Step 4 — Paste, edit, upload

  1. Open the sketch from the previous step (the auto-watering version).
  2. Edit the four config strings at the top: WIFI_SSID, WIFI_PASSWORD, BOT_TOKEN, CHAT_ID.
  3. Click the tick (✓) icon to verify it compiles.
  4. Click the arrow (→) icon to upload. You'll see Connecting... then a progress bar.
  5. No buttons to press. The NodeMCU's USB-serial chip auto-enters flash mode via DTR/RTS. If upload fails with espcomm_open failed, close any other program holding the COM port (PuTTY, Serial Monitor on another window).

Step 5 — Watch it run

  1. Open Tools → Serial Monitor, set baud rate 115200.
  2. You should see Wi-Fi....OK, IP=192.168.x.y then a moisture line every 60 s.
  3. Bench test: hold the probe in air. Within a minute, the relay should click and the pump should run for 5 s. Telegram should ping you afterwards.
  4. Then plant it. Probe into the soil, tube end into the pot, pump body sitting in a glass/bottle of water.

7. Test it online — no hardware needed

You can run the entire firmware in your browser using Wokwi, a free ESP8266 simulator. Wokwi's virtual Wi-Fi reaches the real internet, so the simulated NodeMCU will send real Telegram messages to your phone. Drag a slider to simulate dry soil; watch the relay click on; see the alert arrive.

What's already prepared for you

Wokwi has heavily pivoted to ESP32-first since 2024. Their ESP8266 (Wemos D1 Mini) support still works but is less polished. Two sets of files are ready — try the ESP32 path first, fall back to ESP8266 only if needed. The production firmware that flashes onto your real NodeMCU is unchanged either way.

VariantFilesStatus
A — ESP32 (recommended) diagram_esp32.json, sketch_esp32.ino Most reliable on Wokwi today
B — ESP8266 (fallback) diagram.json, sketch.ino Works but less stable in Wokwi

Important: the ESP32 variant is for simulation ONLY. Your physical hardware is an ESP8266 NodeMCU, so the production sketch (plant_communicator.ino) stays as-is. The simulation is just to validate the logic before you wire anything up.

Launch in 6 clicks — Option A (ESP32, recommended)

  1. Open this direct link: wokwi.com/projects/new/esp32. A blank ESP32 DevKit will appear with two tabs at the top: sketch.ino and diagram.json.
  2. Copy the contents of wokwi/diagram_esp32.json into Wokwi's diagram.json tab, replacing the default.
  3. Copy the contents of wokwi/sketch_esp32.ino into Wokwi's sketch.ino tab.
  4. Click the Library Manager icon in the left sidebar → add UniversalTelegramBot and ArduinoJson (pick version 6.21.x).
  5. In the sketch, edit the BOT_TOKEN and CHAT_ID lines. (SSID is already set to Wokwi-GUEST.)
  6. Click the green ▶ Play button. Serial Monitor will show Wi-Fi OK then sensor readings every 5 seconds.

Option B fallback — ESP8266 / Wemos D1 Mini

Only if Option A doesn't load for some reason. Direct link: wokwi.com/projects/new/wemos-d1-mini. Then paste wokwi/diagram.json and wokwi/sketch.ino (the non-_esp32 files) instead. Same library add and bot token edit as above.

What to do once it's running

  • Drag the soil sensor's slider toward "DRY" (raw value rises past 700). Within ~5 seconds you'll see DRY -> pumping in the Serial Monitor, and the relay module's LED will light up — that's the moment the real pump would turn on.
  • After 3 seconds the burst ends; after 8 more seconds (soak), the sketch re-reads the sensor.
  • If you slid the slider back toward wet during the soak, you'll get the success message on Telegram: "I was thirsty so I watered myself…"
  • If you kept it dry, you'll get the SOS alert: "Tried to water but still dry. Check the tank!"

Once the simulation behaves correctly, flash plant_communicator.ino (the production sketch — longer real-world timings) onto your physical NodeMCU. The logic is identical.

8. If something doesn't work

Upload error espcomm_open failed
Wrong COM port, or the Serial Monitor on another Arduino window is holding it. Close it, re-select COM, retry.
Board not detected at all
Wrong cable (charge-only). Try a different Micro-USB cable, ideally one you know carries data.
Pump runs at boot for a moment
You have an active-HIGH relay. Set PUMP_ACTIVE_LOW = false in the sketch and re-flash.
Pump never runs, but Serial shows "DRY -> pumping"
(a) No common ground between PSU and NodeMCU. (b) External PSU is dead or wired backwards into NO. Multimeter the NO terminal voltage with relay engaged — should read full PSU voltage.
NodeMCU resets when pump starts
You're sharing the NodeMCU's USB rail with the pump. Don't. Move the pump onto its own PSU.
Telegram never receives anything
(a) You forgot to send the bot a message first — bots can't initiate chats. (b) Wrong CHAT_ID. Visit https://api.telegram.org/bot<TOKEN>/getUpdates after messaging the bot and copy the id.
Sensor reads constant 0 or 1023
Loose AOUT wire, or sensor not powered. Multimeter the 3.3 V rail at the sensor's VCC pin.
